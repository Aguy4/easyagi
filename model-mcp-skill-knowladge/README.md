## 视频简介

这期视频介绍通用智能中的 MCP Server、Skill、大模型、知识库和 Token 用量统计 5 个模块。前四个均为流程开发中的**大模型节点**和**角色定义**提供配置数据；第五个用于用量监控。

## 模块说明

### 1. 大模型提供商

配置各厂商大模型的连接信息。

**操作步骤：**

1. 进入大模型界面，点击「新建」
2. 输入自定义模型名称，选择提供商（支持通义千问、智谱、DeepSeek、Kimi 及本地模型）
3. 填写 API Key 与模型名称；若支持图片/视频，勾选多模态
4. 填写备注后保存

### 2. MCP Server

配置供大模型调用的 MCP Server。

**操作步骤：**

1. 进入 MCP Server 界面，点击「新建」
2. 输入名称，选择协议（stdio、SSE、streamable HTTP）
3. 按协议填写对应配置
4. 系统运行在 Docker 中时，调用物理机 stdio 协议 MCP 需**开启代理**
5. 保存

### 3. Skill

为大模型提供 Skill 配置。

**操作步骤：**

1. 进入 Skill 界面，点击「新建」
2. 输入 Skill 名称
3. **安装命令**与**上传压缩包**二选一（安装命令仅支持 SkillHub）
4. 填写功能描述后保存

### 4. 知识库

配置供大模型检索的知识库。

**操作步骤：**

1. 新建知识库，填写名称与备注
2. 在文档列表上传 PDF、Word、TXT 等格式文档

### 5. Token 用量统计

统计大模型 Token 消耗，支持多维度查看：

- 大模型类别
- 输入 / 输出
- 当天
- 近 7 天

## 与工作流的关系

上述四个配置模块（大模型提供商、MCP、Skill、知识库）在搭建工作流时，会被大模型节点和角色定义直接引用，是开发智能体应用的前置准备工作。
