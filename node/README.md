## 视频简介

本期系统介绍开发过程中常用的**工作流节点类型**及**流程连接规则**，是掌握平台可视化编排的基础教程。

## 流程基本规则

- 每个流程必须包含**开始类节点**以触发执行
- 开始节点类型：命令、飞书消息、定时器、图片/文字/数字输入等
- 每个节点均配有帮助与配置说明
- **强数据类型**：仅相同类型的输入输出可直连
- 支持的变量类型：字符串、数字、布尔、结构体、图片、数据集、模型
- **灵活连接**：数字、文字、布尔、对象类型输出可连接至字符串类型输入

## 节点分组说明

### 控制分组

| 节点 | 作用 |
|------|------|
| 变量 / 结构体 | 存放过程数据；支持同名同实例（多节点共享同一实例） |
| 计数器 / 条件 / 选择 | 流程分支控制 |
| JSON 解析 | 通过 jq 语法解析 JSON |
| 字符串拼接 | 拼接字符串 |
| 编程 | 处理复杂数据逻辑 |
| 类型转换 | 在不同变量类型间转换，兼顾严谨与灵活 |

### 输出分组

| 节点 | 作用 |
|------|------|
| 结果 | 定义命令返回值，可配置结果属性 |
| 飞书发送消息 | 将内容发送给指定飞书用户 |
| 输出图片 / 文本 / 数字 | 调试展示或界面开发锚点 |

### 人工智能分组

| 节点 | 作用 |
|------|------|
| 问题分类 | 大模型分类输入，单路输出并透传原内容 |
| 任务分配 | 大模型按角色拆分任务，每角色任务内容不同 |
| 消息路由 | 多 Agent 场景下的消息路由 |
| YOLO8 | 机器视觉，容器本地运行 |
| OCR | 文字识别，容器本地运行 |
| 大语言模型 | 执行复杂推理与工具调用 |

### 工具分组

| 节点 | 作用 |
|------|------|
| 截图 | 截取代理所配置软件所在电脑的屏幕 |
| 邮件 | 发送邮件 |
| 数据库 | 读写数据库 |
