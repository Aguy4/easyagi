## 视频简介

多 Agent 协作，模仿公司组织架构。

演示更灵活的**多 Agent 协调**案例：1 个主 Agent 指挥 3 个次 Agent，用户既可对话总经理，也可分别 @ 各角色 Agent，适用于企业内多角色分工场景。

## 角色设定

| Agent | 角色 | 能力 |
|-------|------|------|
| 主 Agent | 公司总经理 | 指挥下属、对接用户；默认处理未 @ 的消息 |
| 次 Agent 1 | 公司人事 | 人事相关任务 |
| 次 Agent 2 | 公司会计 | 财务相关任务 |
| 次 Agent 3 | 公司法务 | 法务相关任务 |

四个 Agent 均使用**角色定义**功能，具备长期记忆并可持续优化能力（角色定义详见其他专题视频）。

## 消息路由设计

工作流使用 **消息路由节点**，连接关系如下：

### 用户 ↔ 4 个 Agent

- 用户可联系全部 4 个 Agent
- **总经理为默认节点**：消息未 @ 特定智能体时，由总经理处理

### 总经理

- 可联系：用户 + 人事、会计、法务三个次 Agent

### 三个次 Agent

- 可联系：用户 + 总经理

## 设计说明

| 方面 | 说明 |
|------|------|
| 消息路由节点 | 相对旧版降低多 Agent 开发复杂度 |
| 中间过程可见性 | 可将各 Agent 的思考/中间步骤连接给用户，便于调试与透明化 |
| 飞书身份变量 | 专用变量存放飞书发送者身份信息，用于路由与权限逻辑 |

## 与 demo/2 的对比

| 维度 | demo/2（股票系统） | demo/3（公司协作） |
|------|-------------------|-------------------|
| 协作模式 | 命令调用 + 结果汇总 | 消息路由 + @ 提及 |
| 交互入口 | 飞书任务 / 命令 | 飞书群聊式多角色对话 |
| 典型场景 | 专业分析流水线 | 组织内分工协作 |
