## 视频简介

介绍由 **4 个 Agent** 协作完成的**股票交易建议系统**：针对指定股票，从技术分析、新闻事件、综合分析等不同维度给出建议，并由最终决策者汇总后通过飞书反馈给用户。

## 系统架构

```
                    ┌─────────────────┐
                    │   最终决策者     │
                    │  （汇总 + 飞书）  │
                    └────────┬────────┘
                             │ 调用「获取交易建议」命令
         ┌───────────────────┼───────────────────┐
         ▼                   ▼                   ▼
┌────────────────┐ ┌────────────────┐ ┌────────────────┐
│  技术分析型     │ │ 新闻事件驱动型  │ │  综合分析型     │
└────────────────┘ └────────────────┘ └────────────────┘
```

| 角色 | 职责 |
|------|------|
| **最终决策者** | 汇总三个子 Agent 建议；查询最新新闻、近 7 天新闻与成交信息；输出最终交易建议并飞书通知用户 |
| **技术分析型** | 按提示词侧重技术面分析 |
| **新闻事件驱动型** | 侧重新闻与事件驱动 |
| **综合分析型** | 综合多因素分析 |

## 技术要点

### 子 Agent（3 个）

- 使用**单次对话接口**，不记录会话历史，避免影响后续判断
- 启用**轻装模式**
- 均挂载：**新闻搜索 MCP** + **股票信息搜索 MCP**
- 开启**深度思考**
- 通过**字符串拼接节点**将建议汇总后返回给最终决策者

### 最终决策者

- 需能调用「获取交易建议」命令 → 为该命令添加清晰**描述**，便于大模型正确选型

## 适用场景

多角色、多视角的专业分析类应用；需要在汇总层融合外部实时数据（新闻、行情）的智能体系统。
